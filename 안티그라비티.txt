지하철 도착 정보를 확인하는 반응형 웹페이지를 개발할거야. 
@design 디렉토리의 디자인 정보를 참고해서 구현해줘. 다음의 요소들을 고려해야해.
#기능
- 사용자가 지하철역 이름을 입력하면 "서울시 지하철 실시간 도착정보" API를 통해서 도착예정 정보를 조회해.
- 한번 입력하고 정상적으로 조회된 지하철역 정보는 최근 5개까지 아래 버튼 형식으로 표시해서 입력을 선택할 수 있어.
- 검색결과에 따라 정상, 비정상 결과와 조회 시간을 간략히 표시해주고,
- "지하철호선"과 "도착지-다음역"을 정보로 묶어서 카드 형식으로 나열해 보여주고,
- 각 카드안에는 "도착 예상 시간", "현재 위치", "열차번호" 목록을 표시해줘.
- 카드 안의 목록은 최대 3개야. 목록 옆에는 '선택' 버튼을 넣어줘. 이 버튼은 나중에 따로 구현할거야.
#개발환경
- 프레임워크는 Next.js 를 사용해. (App Router 방식 사용)
- 언어는 TypeScript 사용해.
- 클라이언트에서 외부 API 직접 호출 금지. Next.js의 Route Handlers (app/api/...)를 만들어서 백엔드 프록시 형태로 구성할 것.
- 모든 외부 API 키는 .env.local 파일에서 환경 변수로 관리할 거야.
- 서울 지하철 정보 API 연동을 위한 키는 .env.local 안에 SEOUL_SUBWAY_API_KEY 이름으로 주어질거야.

아래 내용을 수정해줘 .

- 문자열 "© 2024 Silent Conductor Transit Systems"를 "© 2026 by doohee2"로 변경해줘.
- "시스템 맵"을 누르면 "http://www.seoulmetro.co.kr/kr/cyberStation.do" 주소로 새창을 열어주도록 해.

지하철 도착 정보 조회 후, 결과 카드 목록에서 열차번호 '선택' 버튼을 눌렀을 때, 열차의 위치 정보와 경로를 보여주는 웹페이지를 추가할거야.
@design/code2.html 과 @design/screen2.png 파일을 참고해서 구현해주고, 다음의 기능들을 고려해야해.
#기능
- 맨 위에는 선택한 열차의 "지하철호선"과 "도착지-다음역", "열차번호"와 "현재역"이 보이고, 옆에는 "다시 조회" 버튼이 있어.
- "다시 조회" 버튼은 나중에 구현할 거야.
- 지하철 노선 방향에 따라 현재 위치부터 종점까지의 열차 목록을 위에서부터 아래로 연결된 노드 형태로 보여줘.
- 노선에 대한 정보는 @json/"서울교통공사_노선별 지하철역 정보.json" 파일을 참고해서 일치하는 호선과 지하철역 리스트를 만들어서 구현하면 돼.
- 선택한 열차의 상행/하행 방향에 따라서, 지난 역이 맨 윗줄의 첫번째 노드이고, 두번째 줄이 현재역, 세번째 노드부터 다음 도착역이 보이는거야.
- @json/"서울교통공사_역간거리.json" 파일을 참고해서 현재역으로부터 소요시간을 구할 수 있는 경우에는, 역명 옆에는 버튼 형태로 예상 도착시간을 표시해줘.
- @json/"서울교통공사_역간거리.json" 파일에서 현재역 정보를 찾을 수 없는 경우는, 버튼에 내용을 표시하지 말고 비활성화된 형태로 만들어줘.
- 역명 옆 "버튼"의 이름은 "역명 버튼"이고, 이 기능도 나중에 구현할거야.
구현 계획을 먼저 작성해서 확인하고 작업해줘.
